// 01 — Orbital Mechanics

The LGO Concept

An LGO is an elliptic orbit whose apogee velocity exactly matches the velocity of a point on Earth's surface at a given latitude — making the satellite appear nearly stationary from that region during its apogee passage.

🛰️

Special Case (φ = i)

When the sub-satellite latitude at apogee equals the inclination, the argument of perigee is fixed at ω = 90° or 270°. Classic Tundra and Molniya-family orbits fall into this category.

φ = i
🌐

General Case (φ ≠ i)

When |φ| < |i|, a broader family is unlocked. The argument of perigee ω is computed from formula (23) of Razoumny [2018], completing the orbital elements {a, e, i, ω} for the orbit propagator.

φ ≠ i, |φ| < |i|
🔁

Repetition Factor m/n

The ratio m/n (m nodal revolutions in n efficient astronomical days) parameterises the ground-track repeat cycle. J2 nodal precession is included in the computation via the K correction factor.

J2 corrected
⏱️

Apogee Dwell Time

The key performance metric is Δt — time spent within the true anomaly window θ ∈ [170°, 190°]. LGO variants improve dwell time by 33–35% over classical Tundra and Raindrop baselines.

Kepler's eq.
📡

Orbit Propagation

Output orbital elements {a, e, i, ω} feed the R2BP Orbit Propagator developed in the MSc thesis [Luciardello 2024], generating ground tracks and ECI J2000 trajectories over 10-day windows.

R2BP propagator
📈

Velocity Curves

The tool plots circular velocity V_c(H_A), apogee velocity V_A at fixed perigee altitudes (500, 5 000, 15 000 km), and the Earth-surface velocity V_i(H_A/i) as functions of apogee altitude up to 60 000 km.

// 02 — Development

From MATLAB to C

The tool was developed in three stages, each adding precision and usability without sacrificing the validated physics from the previous stage.

Phase 1 · 2024

MATLAB Prototype

The analysis was originally scripted in MATLAB®, replicating Fig. 6 of Razoumny [2018]. Apogee radius r_A was located via a search-grid method: a dense altitude grid (0–60 000 km) was scanned point-by-point to find where the computed m/n matched each target value. Results were validated against published figures for the special case (i = 0°, 63.4°) and the general case.

method: grid search over H_A ∈ [0, 60000] km
cases: special (φ=i) · general (φ≠i)
output: V_A curves, m/n curves, orbit markers
ref: Fig.6 of Razoumny 2018 [reproduced]

Phase 2 · 2024

C Implementation (LGO_C2 / LGO_GUI)

The MATLAB script was re-implemented in C for portability and to add a GUI. The core computation — grid evaluation of m/n over the full altitude range — was preserved verbatim, with the addition of J2 nodal correction and cross-platform build support (Linux and Windows/MSYS2). Results are written to a structured text table and passed to Gnuplot for live rendering.

grid: N_H = 60001 points, 1 km resolution
perigee alts: 500, 5000, 15000 km
output: data/table_results.txt + gnuplot
platforms: Linux · Windows (.exe)

Phase 3 · 2024

Newton-Raphson Solver (lgo_newton.c)

A Newton-Raphson root-finding module replaced the grid-search marker-identification step. The solver brackets the root with a 30-point log-spaced coarse scan, then refines with Newton steps using a central-difference numerical derivative. A bisection fallback (Brent-style) activates whenever a Newton step escapes the bracket or lands in a numerically flat region. Convergence is declared at |m/n(r_A) − target| < 10⁻⁹. Validation confirmed no significant difference in computed altitudes versus the grid approach, while eliminating grid-resolution artefacts.

tol: 1e-9 · max_iter: 80 · scan pts: 30
deriv: central difference, h = r_A·1e-5
fallback: bisection (Brent-style)
result: no significant Δ vs. grid search ✓

Phase 4 · 2024

GTK3 GUI Front-end

A GTK3 graphical interface wraps the computation layer, allowing full parameter configuration without recompilation. The GUI exposes all inputs — inclinations, m/n targets, special vs. general case toggle, ω or φ entry — and displays the results table inline after each run.

framework: GTK3 · libm · Gnuplot
inputs: I[], mn0[], case, ω or φ
outputs: {a, e, i, ω} table + live plot
note: linux version faster than win32

// 04 — Build & Run

Compile the Tool

The tool compiles to a single binary (GUI + computation) with no runtime dependencies beyond GTK3 and Gnuplot. The Newton solver is a drop-in include with its own standalone test target.

Linux 
# Install dependencies
sudo apt install libgtk-3-dev gnuplot

# Compile GUI
gcc -O2 -Wno-deprecated-declarations \
    $(pkg-config --cflags gtk+-3.0) \
    -o lgo_gui LGO_GUI.c \
    $(pkg-config --libs gtk+-3.0) -lm

# Run
./lgo_gui
Windows (MSYS2 MinGW64) 
# Install dependencies
pacman -S mingw-w64-x86_64-gcc \
  mingw-w64-x86_64-gtk3 \
  mingw-w64-x86_64-gnuplot

# Compile GUI
gcc -O2 $(pkg-config --cflags gtk+-3.0) \
    -o lgo_gui.exe LGO_GUI.c \
    $(pkg-config --libs gtk+-3.0) -lm -mwindows

# Run
lgo_gui.exe
Newton Solver — Standalone Test 
# Compile test binary
gcc -O2 -DLGO_NEWTON_TEST \
    -o lgo_newton_test lgo_newton.c -lm

# Run solver validation
./lgo_newton_test

# Output: H_A, Va, e for all test cases
# Cases: i=0,63.4 deg / special+general
Integrating the Newton Solver 
// Add at top of LGO_C2.c or LGO_GUI.c:
#include "lgo_newton.c"

// Replace find_hindex() calls with:
newton_solve_all(mn0, n_mn0,
    i_rad, i_rad, 1,   /* special */
    rA_s, Va_s, rP_s);

// General case (phi from omega):
newton_solve_all(mn0, n_mn0,
    i_rad, phi, 0,      /* general */
    rA_s, Va_s, rP_s);

// 05 — Source Files

Repository Contents

Two self-contained C files — the GUI application and the Newton solver module.

.c

LGO_GUI.c

Main application. Contains the full physics computation (V_A, V_i, m/n grids, J2 correction, orbital element derivation for special and general cases) and the GTK3 GUI layer. Drives Gnuplot for live plotting and writes results to data/table_results.txt. Compiles to a single binary for Linux or Windows.

.c

lgo_newton.c

Drop-in Newton-Raphson solver module. Replaces the find_hindex() grid-search call with a root-finding approach: 30-point coarse scan → Newton step with central-difference derivative → bisection fallback. Convergence tolerance 10⁻⁹. Includes a standalone test harness activated via -DLGO_NEWTON_TEST.

.md

README.md

Full project documentation: background physics, development history, build instructions for Linux and Windows, API reference for the Newton solver, results summary, and physical constants used throughout the computation.
